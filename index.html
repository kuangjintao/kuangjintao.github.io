0x00 CentOS中安装python3.10

在CentOS中安装较新版本的python。

由于较低版本的openssl存在漏洞，所以较新版本的python需要较高版本的openssl作为支撑，否则不会安装_ssl模块

0x01 安装OPENSSL

  1. 下载openssl源码
  2. 解压至/usr/local/src/，命令为tar -xf openssl-1.1.1q.tar.gz -C /usr/local/src/
  3. 在解压目录进行编译安装，指定安装目录为/usr/local/openssl
  1. （可选）在/usr/lib64下创建软连接
  1. （可选）替换掉$PATH中的openssl，执行openssl version -a验证安装情况

0x02 安装python

  1. 下载源码、解压，编译安装
  1. 修改/usr/bin下的软链接

0x03 修改yum配置文件

  1. 如果将/usr/bin/python指向python3的话，需要修改yum配置文件，因为yum使用的是python2

0x00 漏扫授权破解

虽然有NESSUS、AWVS可用，但为什么不支持一波国货呢？

0x01 安装漏扫环境

  1. 在VMWare上安装漏扫产品
  2. 本机新增一块Microsoft Loopback Adapter，并为之设置合适的IP地址
  3. VMWare上设置漏扫产品的网络连接，使之桥接至新增的环回网卡
  4. 在本机浏览器上访问漏扫网址

注：这种网络连接方式无需修改漏扫产品的IP，即可在本机访问漏扫web页面

0x02 找寻关键代码

  1. 开启漏扫的SSH、SFTP服务
  2. 连接SFTP，下载并反编译漏扫的web程序
  3. 在本机上抓取上传授权文件的HTTP流量
  4. 结合web框架，在反编译后的代码中找出上传授权文件对应的函数

0x03 破解方式1：绕过验证

  1. 一层一层函数地追踪，寻找验证授权文件的逻辑分支
  2. 更改代码的判断逻辑。对于java代码，可以修改java文件的逻辑，再重新编译为class文件，也可以使用asmtools直接修改class文件中的逻辑；对于dll文件或so文件，通过在IDA中修改跳转方式，或修改关键寄存器的值，来修改判断逻辑
  3. 用修改后的文件替换掉漏扫中原有的文件
  4. 重启漏扫，上传文件触发授权验证

0x04 破解方式2：生成授权文件

  1. 一层一层函数地追踪，寻找解密授权文件的算法
  2. 逆向操作，编写生成授权文件的代码。可以将原java代码引用为依赖，或在自己的代码中加载dll、so文件，来复用一些有用的函数
  3. 生成并上传授权文件

0x00 设置博客图床

使用github仓库存储图片

0x01 新建图片仓库

  1. github新建仓库，比如新建一个Picture仓库
  2. 在账号的Settings中生成一个新的Personal access tokens，记作token string

0x02 使用picgo

  1. vscode安装PicGo插件
  2. 设置扩展，使图片上传目的地为github仓库

  1. 编辑md时，使用Ctrl+Alt+E上传本地图片，或使用Ctrl+Alt+U上传剪切板图片

注：无法加载图片的话，可在本地HOSTS文件中添加185.199.108.133 raw.githubusercontent.com

0x00 安装hexo

在centos上安装hexo，并将文章发布至github-pages

0x01 运行环境 && 本地安装

  1. centos下载node.js安装包，解压并重命名目录，比如/root/node.js
  2. 编辑/etc/profile文件，配置环境变量
  1. 运行环境准备就绪，使用npm安装hexo
  1. 创建博客目录、初始化、启动

0x02 git配置 && 发布至github

  1. 注册github，假设注册邮箱为A@EMAIL.com，用户名为U
  2. 添加Repository，仓库名为U.github.io
  3. 在该仓库新建一个index.html，浏览器访问U.github.io查看效果
  4. centos上配置git，并生成ssh密钥对
  1. 确认/root目录下生成了 .gitconfig 与 .ssh ；将生成的公钥添加至github的SSH keys
  2. 测试git配置是否成功
  1. 安装hexo发布插件
  1. 修改hexo的根目录配置文件/hexo/_config.yml，添加部署仓库
  1. 发布到github

0x03 第一篇文章

编辑完文章后，重新部署

0x04 修改主题

  1. 下载主题至/hexo/themes，如/hexo/themes/next
  2. 必要情况下，cd /hexo && npm i更新依赖
  3. 修改/hexo/_config.yml中的theme，如theme: next
